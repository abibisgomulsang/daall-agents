# 멀티 에이전트 3라운드 토론 — 직원별 모델/페르소나/메모리

## 사장님 요청 그대로

> Ollama로 무료 기본, 각 직원에게 개별 모델/페르소나/메모리 부여 —
> 데이터/검수는 Claude, 마케팅은 ChatGPT, 그 외는 Ollama.
> 1라운드 의견 → 2라운드 반박/동조 → 3라운드 CEO 종합.
> CEO가 LLM으로 작업 분배. 의견 충돌 시 표결/CEO 판단.

## 구조 한눈에

```
│  · 09_approval 흐름을 결론에 자동 명시             │
└──────────────────────────────────────────────────┘
```

## 직원별 매핑

| 직원 | 모델 | 시스템 프롬프트 핵심 |
|---|---|---|
| 데이터 AI | **Claude** | 노출/CTR/CPC/ROAS 관점, 감정 없는 표·숫자 |
| 검수 AI | **Claude** | 과장광고/저작권/개인정보/브랜드 톤 |
| 마케팅 AI | **OpenAI ChatGPT** | 후킹 2초·구매 전환 카피 |
| 고객심리 AI | Ollama gemma4:latest | 집사 고민·감정·구매 동기 |
| SNS AI | Ollama gemma4:e2b | 릴스 후킹·자막·썸네일 |
| 상품 AI | Ollama gemma4:latest | 옵션·세트·리필·재구매 |
| 네이버광고 AI | Ollama gemma4:latest | 키워드·낭비 클릭·ROAS |

각 직원은 자기 페르소나·말투·메모리 폴더(`11_memory/agents/<dir>/rounds.jsonl`)를 가집니다.
**프롬프트 본문은 저장되지 않고**, 라운드 번호·길이·제공자만 기록됩니다.

## 비용 보호 (자동)

회의 시작 전에 직원 수 × 라운드 수 × 평균 토큰으로 추정 비용을 계산하고
`usage_caps.check_cap()`로 일일/월 캡(기본 ₩1,000/일, ₩30,000/월)을 검사합니다.
초과 위험이면 **자동으로 모든 직원을 Ollama로 강등**(`force_ollama=True`).
사장님이 신경 안 쓰셔도 돈 새는 일은 없습니다.

추정 비용 예시 (회의 1회):

| 모드 | 추정 |
|---|---|
| 모두 Ollama (force_ollama) | ₩0 |
| 데이터·검수=Claude, 마케팅=OpenAI, 나머지 Ollama | ₩100~200 |
| 전부 Claude (특수 회의) | ₩500~800 |

## 명령

```powershell
cd D:\AI_COMPANY
.\.venv\Scripts\Activate.ps1

# 단독 멀티 회의
python -m ai_company.main multi-meeting --topic "네이버광고 ROAS 1.4 회복"

# 외부 API 안 쓰고 Ollama만 (무료)
python -m ai_company.main multi-meeting --topic "고스틱 광고 후킹 전략" --no-live

# boss-command 안에서 멀티 회의 사용 (7단계 흐름 + 3라운드 토론)
python -m ai_company.main boss-command --message "고스틱 광고 효율 개선" --live --multi
```

회의록은 `10_meetings/multi_meeting_*.md` 와 같은 디렉터리의 `.json` 으로 저장.
각 직원 발언은 `11_memory/agents/<dir>/rounds.jsonl` 에도 누적.
