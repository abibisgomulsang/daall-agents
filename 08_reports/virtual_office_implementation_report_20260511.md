# 가상 사무실 페이지 구축 보고서

## 1. 수행한 일

사장님이 보내주신 가상 사무실 스크린샷 4장(메인 / 활동 / 산출물 / 대화록)을
그대로 재현한 **가상 사무실** 페이지를 만들었다.

- 상단 상태바: 에이전트 팀 · DAY · TIME · 24시간 자동 운영 ON/OFF ·
  직원 에이전트 보기 링크 · 설정/로그/파일 아이콘
- "사무실 가동 중 · [에이전트A] → [에이전트B] (잡담/회의)" 실시간 표시
- SVG 사무실 평면도 — 7개 방(리서치룸/스튜디오/대회의실/복사실/휴게실/운영실)
  과 책상 + 캐릭터(에이전트 컬러 점 + 이름 두 글자 라벨)
- 우측 사이드: **활동 / 산출물 / 대화록** 3탭
- 자율 모드 ON 상태에서 1.5초마다 한 틱 진행 — 시간 5분씩 흘러가며 페어 잡담 /
  회의 / 이동 / 산출물 생성 이벤트가 누적됨
- 가끔 LM Studio 미실행 mock 에러 메시지도 자연스럽게 끼움 (사장님 스크린샷 4번 톤)

## 2. 생성 파일

`06_apps/virtual_office/`

- `index.html` — 상단 상태바 + 평면도 + 3탭 + 푸터
- `styles.css` — 다크 테마 (사장님 스크린샷의 검정 + 형광초록 톤)
- `app.js` — 시뮬레이션 엔진, SVG 맵 그리기, 탭 전환, 자율 모드 토글
- (`agents_data.js`는 `../agent_matrix/agents_data.js`를 그대로 재사용 — 단일 진실 소스 유지)

## 3. 시뮬레이션 동작

매 1.5초마다:

| 확률 | 이벤트 | 동작 |
|---|---|---|
| 45% | 페어 잡담 | 두 에이전트가 휴게실/복사실로 이동 후 점선으로 연결 |
| 25% | 회의 | 두 에이전트가 대회의실 테이블에 모임 |
| 15% | 자리 이동 | 한 에이전트가 자기 책상으로 복귀 |
| 15% | 산출물 생성 | "릴스 대본 3안", "썸네일 시안 2종" 같은 결과 누적 |
| 4% (보너스) | LLM 에러 mock | 대화록에 빨간색 경고 메시지 (스크린샷 4번 톤) |

시간이 24:00을 넘으면 DAY +1 후 09:00으로 리셋. 활동/대화록/산출물은
각각 60/80/20개까지 누적 후 오래된 항목 자동 폐기.

## 4. UI 매칭 (사장님 스크린샷 ↔ 결과)

| 스크린샷 요소 | 본 페이지 위치 |
|---|---|
| `나만의 에이전트 팀` 타일 | 좌상단 첫 타일 |
| `DAY 1` 빨간 점 | 두 번째 타일, 자동 증가 |
| `TIME 14:20` | 세 번째 타일, 자율 모드 ON 시 5분씩 진행 |
| `24시간 자동 운영 ON` 큰 초록 박스 | 네 번째 타일, 클릭으로 토글 |
| `직원 에이전트 보기` 초록 박스 | `../agent_matrix/index.html` 이동 |
| `사무실 가동 중 · Designer — Designer → 복사실` | strip 영역에 실시간 갱신 |
| 픽셀아트 사무실 맵 | SVG 평면도 (캐릭터 컬러 점 이동) |
| `활동 / 산출물 / 대화록` 3탭 | 우측 사이드 패널 |
| `아직 산출물이 없어요` 빈 상태 | 산출물 탭 초기 화면 |
| `## [14:18:12] CEO · _작업 분배_` 마크다운 | 대화록 탭의 누적 로그 |
| 빨간 LLM 호출 실패 메시지 | 4% 확률로 mock 표시 |

## 5. 사용 방법

```powershell
# 가장 빠름: 파일 더블클릭 또는
start D:\AI_COMPANY\06_apps\virtual_office\index.html

# 서버 켰을 때
http://127.0.0.1:8765/06_apps/virtual_office/index.html
```

기존 `agent_matrix`와 `ai_office_simulator` 두 페이지에서도 "가상 사무실"
링크가 있어 한 번 클릭으로 이동 가능.

## 6. 확장 포인트

- 캐릭터 외형: 현재는 컬러 원 + 이름 2글자. PNG 스프라이트 추가 시 픽셀아트로 업그레이드 가능
- 백엔드 연결: 자율 모드 이벤트를 `local_viewer_server`에 POST 해서 활동 로그를
  `12_logs/`에 영구 저장 (현재는 브라우저 메모리)
- 진짜 라우터 호출: 사장님 명령이 들어오면 `nl_command.interpret()` → 해당
  에이전트가 작업하는 식으로 실제 dry-run 산출물 생성과 연결

## 7. 사장님 확인 필요한 부분

- 에이전트 컬러/맵 레이아웃 (현재 색은 `agents_data.js`의 cardFrom 사용)
- 자율 모드 틱 간격 (현재 1.5초 = 가상 5분)
- 산출물 종류 8개 — 추가/삭제 원하시면 알려주세요

## 8. 안전 확인

- 외부 API 호출 / 모델 다운로드 / 발송 — 없음
- 자율 모드 시뮬레이션은 브라우저 메모리에만 존재 (페이지 새로고침 시 초기화)
- 실제 광고/스마트스토어/SNS 반영은 어떤 시뮬 이벤트가 발생해도 일어나지 않음
- 푸터에 "모든 활동은 dry-run 시뮬레이션입니다" 명시
