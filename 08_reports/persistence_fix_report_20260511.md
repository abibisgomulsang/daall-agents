# 페이지 종료 시 초기화 문제 — 영속화 수정 보고서

## 1. 사장님이 보고하신 증상

> 윈도우창을 종료했다가 다시 하면 초기화가 되어 있다.

브라우저 페이지를 닫거나 새로고침하면 DAY/TIME, 활동 로그, 대화록, 산출물,
채팅 내역이 모두 초기화되던 문제 — **세 페이지 모두 localStorage에
자동 저장하도록 수정**했습니다.

## 2. 페이지별 적용 내역

### 가상 사무실 (`06_apps/virtual_office/app.js`)

저장 항목:
- DAY, TIME(분)
- 자율 모드 ON/OFF
- 활동 로그 최근 40개
- 대화록 최근 60개
- 산출물 최근 20개

저장 시점:
- **매 1.5초 틱 후** (자동)
- 자율 모드 토글 시
- 페이지 종료 직전 (`beforeunload`)

복원: 페이지 로딩 시 자동 → "이전 상태 복원됨 (DAY 3 14:25)" 로그 표시

초기화: ⚙ 아이콘 **더블클릭** → 확인창 후 모두 삭제

저장 키: `ai_company.virtual_office.state.v1`

### AI 사무실 시뮬레이터 (`06_apps/ai_office_simulator/app.js`)

저장 항목:
- 채팅 메시지 최근 60개 (역할/본문/메타/시각)

저장 시점:
- 채팅 메시지가 추가될 때마다 자동

복원: 페이지 로딩 시 자동 → "이전 대화 N개 복원됨" 안내

초기화: 채팅창 빈 영역 **더블클릭** → 확인창 후 삭제

저장 키: `ai_company.simulator.chat.v1`

### 직원 에이전트 매트릭스 (`06_apps/agent_matrix/app.js`)

저장 항목:
- 모델 오케스트레이션 매핑 (이미 있었으나 키 이름 정리)
- 현재 필터 (전체/활성/오프라인/채용 대기) — **신규 영속화**

저장 시점:
- 모델 매핑 변경/저장 버튼 클릭 시
- 필터 칩 클릭 시

초기화: "에이전트 매트릭스" 타이틀 **더블클릭** → 확인창

저장 키:
- `ai_company.agent_matrix.model_map.v1` (구 키 `ai_company_model_map`도 마이그레이션)
- `ai_company.agent_matrix.filter.v1`

## 3. 안전 가드

- 모든 `localStorage` 호출은 try/catch — 사용자가 사생활 보호로 막아둔
  브라우저에서도 안전하게 무시
- 저장 데이터 상한 (활동 40 / 대화 60 / 산출물 20 / 채팅 60) — 용량 폭주 방지
- 다른 도메인/페이지로 데이터 누출 없음 — 브라우저 localStorage는 자동 격리
- 초기화는 항상 `confirm()` 확인 후만 진행

## 4. 사장님 확인 방법

```text
1. 가상 사무실 페이지 열기
2. 활동 로그가 몇 개 쌓일 때까지 대기
3. 페이지 닫기 또는 새로고침
4. 다시 열면 "이전 상태 복원됨" 로그가 보이고, DAY/TIME과 활동 로그가 그대로 남아 있어야 함
```

채팅도 마찬가지로 새로고침 후에도 이전 대화가 보입니다.

## 5. 초기화 방법 (정리)

| 페이지 | 초기화 방법 |
|---|---|
| 가상 사무실 | 상단 ⚙ 아이콘 **더블클릭** |
| 채팅창 | 채팅 영역 빈 곳 **더블클릭** |
| 직원 매트릭스 | "⬢ 에이전트 매트릭스" 타이틀 **더블클릭** |

각각 확인창이 떠서 실수 방지.

## 7. 안전 확인

- localStorage 데이터는 사장님 브라우저 안에만 — 외부 전송 없음
- 비밀번호/API 키 같은 민감 정보는 처음부터 저장 대상 아님
- 초기화는 사장님 확인창 후만
- CLI / 백엔드 로직은 일절 변경 없음 — UI 영속화만
